xDB Replication Server in multi-master configurations provides enterprise-class solutions for database replication, backup, disaster recovery, read-scalability and write availability.

Use multi-master replication for read scalability and write availability for geo-dispersed databases. Replace a single overloaded master that consolidates regional data with multiple masters; one in each region that improve local write performance while providing consolidated data access in near real time.

When integrated into a high availability configuration, multiple masters insure that all locations have a master available should one location fail.

Multi-master replication also employs Uniqueness, Update & Delete conflict detection and provides multiple conflict resolution strategies: Earliest Timestamp, Latest Timestamp, Node Priority or manual options. You can even have a standby resolution should your first choice not resolve the conflict.

    PostgreSQL is the world's most advanced open source relational database!

    PostgreSQL has a mature code base in development since 1995 shepherded by a large, vibrant and independent community whose development has accelerated in recent years.

    Born from the same IBM research as Oracle and DB2, it also contains comparable enterprise class features such as full ACID compliance for outstanding transaction reliability, Multi-Version Concurrency Control for access to data under high loads, many performance related features and the best security of any open source database available today.

    PostgreSQL supports standards such as ANSI SQL and SQL/MED and yet is highly extensible with support for over 12 procedural languages, GIN and GIST Indexes, Spatial data support, and multiple NoSQL like features for document or key-value based applications.

                      Postgres Performance Management

                        The EDB Replication Server is a powerful and versatile alternative to PostgreSQL’s all or nothing streaming replication option. Streaming replication copies your entire database and is a great solution for backup configurations and basic read scale out configurations when Hot Standby is employed.

                        EDB Replication Server has many more options available to help solve a wider variety of problems in your data center. For example, for more limited 'data feeder' operations, you can replicate only a single table or a group of tables and each one can filter specific records to be replicated.

                        For more sophisticated solutions EDB Replication Server's multi-master configuration, replaces a single overloaded master that consolidates regional data, with multiple masters in each region improving local write performance while providing consolidated data access in near real time for all locations. When integrated into a high availability configuration, multiple masters insure that all locations have a master available should one location fail.

                        Replication management is also a snap with the easy to use GUI replication console that includes a replication scheduler and history viewer. EDB Replication Server is also integrated into Postgres Enterprise Manager for monitoring and alerts.
